Hardware Implementation

For the hardware implementation, we employed an advanced setup featuring the AS7341 spectral sensor and ESP32 module. The AS7341, a state-of-the-art spectral color sensor, captured detailed light intensity data across various wavelengths. Integrated with the ESP32, our custom-built platform facilitated seamless communication and data transmission. Fruits were strategically placed under controlled lighting conditions to ensure accurate spectral measurements. This hardware configuration provided a robust foundation for acquiring precise spectral color data, vital for training our neural network.

Data Distribution

Our database comprised approximately 3200 rows, encompassing diverse fruits categorized into four distinct classes: Early Ripe, Partially Ripe, Ripe, and Decay. Each class contained a proportional representation of fruits, ensuring a balanced distribution across ripening stages. This diverse dataset facilitated robust model training, enabling our neural network to learn and generalize patterns associated with various stages of fruit ripeness, ultimately enhancing the accuracy and reliability of our fruit ripeness detection system.

Data Collection methodology

The spectral color data for our fruit ripeness detection app was collected using a bespoke hardware setup comprising an AS7341 spectral sensor and an ESP32 module. Fruits were placed in controlled lighting conditions, and the sensor, interfaced with the ESP32, measured light intensity across various wavelengths, capturing detailed spectral information.

The Spectral Color Data

The spectral color data obtained from the AS7341 sensor includes features from F1 to F8, representing distinct wavelength bands. F1 to F8 encompass specific regions of the electromagnetic spectrum, capturing nuanced information about the fruits' spectral characteristics. Additionally, the Near-Infrared (NIR) feature extends the analysis into the near-infrared range, providing valuable insights into molecular vibrations and further enhancing the precision of our fruit ripeness detection system by considering additional spectral signatures beyond the visible spectrum.

Confusion Matrix

A confusion matrix is a vital tool in machine learning evaluation, detailing the performance of a classification model. It displays true positives, true negatives, false positives, and false negatives, allowing for the calculation of metrics like accuracy, precision, recall, and the F1 score. This matrix provides a comprehensive overview of a model's ability to correctly classify instances and identifies areas of strength or weakness, aiding in fine-tuning and improving the overall performance of the machine learning model.

Learning Curve

The learning curve depicts the relationship between a model's training score and the number of instances. As the training set increased from 400 to 1400 instances, the score exhibited notable improvement, rising from 93% to 97%. This upward trend suggests that the model benefits from additional data, showcasing enhanced performance. However, careful consideration is needed to avoid overfitting or underfitting, ensuring optimal balance in model complexity and generalization across varying dataset sizes.

Class Prediction Error

Class prediction errors provide insights into a model's misclassification patterns. In our scenario, for Class 0, 10 instances received inaccurate predictions, for Class 1, nearly 20 test values were misclassified, Class 2 saw around 12 incorrect predictions, and for Class 4, only a few instances resulted in prediction errors. This analysis allows for targeted improvements, focusing on specific classes where the model exhibits challenges and optimizing performance for more accurate predictions.

Results

The initial Artificial Neural Network (ANN) achieved 85% accuracy. Ensembling top-performing models—SVM, KNN, and Naive Bayes—resulted in a powerful Voting Classifier with 96.04% training accuracy and 95.4% precision. KNN led with 93.79%, showcasing effective four-class classification. SVM reached 85.4%, creating a discerning hyperplane, while Naive Bayes achieved 89.94% leveraging probabilistic assumptions. The ensemble model demonstrated robust generalization with 96.04% accuracy on a separate testing dataset.
